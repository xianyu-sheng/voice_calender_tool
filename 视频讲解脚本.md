## 🏗️ 第二幕：项目架构介绍（1-2分钟）

### 2.1 文件结构概览

**【画面】** 打开 GitHub 仓库页面，展示目录结构

**【台词】**

> 首先，我们来看一下项目的整体架构。
>
> 这是一个**前后端分离**的项目，采用 React + Flask 的技术栈。
>
> 项目主要分为三个部分：

**【画面】** 用鼠标依次指向对应文件夹

> - **frontend/** 是前端代码，使用 React 18 + TypeScript 开发，构建工具是 Vite
> - **backend/** 是后端代码，使用 Flask 框架，SQLite 数据库
> - **build.spec** 是 PyInstaller 的打包配置，用于生成独立的 exe 文件

### 2.2 前端结构

**【画面】** 展开 frontend/src 目录

**【台词】**

> 前端的核心代码在 src 目录下：
>
> - **components/** 目录存放所有 UI 组件，包括月视图、周视图、日视图、事件弹窗、任务弹窗等
> - **hooks/** 目录是自定义 Hook，封装了语音识别、语音合成等功能
> - **utils/** 目录是工具函数，包括语音指令解析器和日期工具
> - **App.tsx** 是主组件，管理全局状态和业务逻辑
>
> 前端使用 CSS 渐变主题，支持响应式布局，可以适配桌面、平板和手机。

### 2.3 后端结构

**【画面】** 展开 backend/app 目录

**【台词】**

> 后端采用 Flask 框架，结构清晰：
>
> - **api/** 目录是 API 蓝图，包括事件、任务、日历、提醒、语音等接口
> - **models/** 目录是数据模型，定义了事件、任务、日历等实体
> - **services/** 目录是业务服务，包括 DeepSeek 大模型调用服务
> - 数据库使用 SQLite，轻量级且无需额外配置

### 2.4 技术栈总结

**【画面】** 显示技术栈表格或图表

**【台词】**

> 总结一下技术栈：
>
> | 层级 | 技术 |
> |------|------|
> | 前端 | React 18 + TypeScript |
> | 构建 | Vite |
> | 后端 | Flask + SQLAlchemy |
> | 数据库 | SQLite |
> | 语音 | Web Speech API + DeepSeek |
> | 打包 | PyInstaller |
>
> 这种架构的优点是**开发效率高**、**部署简单**、**易于维护**。
